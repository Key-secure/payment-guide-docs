## I. 功能简介

KYC认证列表模块用于帮助机构管理员实时查看平台会员的身份认证记录及验证详情，追踪各类KYC服务的使用状态，确保平台合规运行及用户身份可审计。

该模块将所有KYC验证历史进行统一展示，支持按用户、状态、时间等维度筛选，并可查看部分认证详情。认证数据来源于您所集成的KYC服务提供商（如Sumsub 或 KeySecure），部分字段仅展示SDK授权范围内的信息，完整数据需前往对应服务平台查看。



## II. 页面说明

| 字段名称 | 描述 |
|------|------|
| 用户ID / 邮箱 | 显示会员唯一标识及注册信息 |
| 提交时间 | 用户发起KYC认证请求的时间 |
| 认证状态 | 当前认证进度状态（如：审核中、已通过等） |


<Note>
每次KYC服务调用（即用户提交验证）都会消耗认证额度，请合理安排调用频次。
</Note>


**查看KYC认证详情**

点击任意认证记录进入详情页，可查看如下字段：

| 字段名称 | 描述 |
|------|------|
| 会员ID | 对应会员的唯一标识 |
| Sumsub验证ID| Sumsub生成的唯一认证流水号 |
| 认证状态 | 与Sumsub接口实时同步的验证状态 |
| 用户姓 / 名| 用户提交的验证的姓名信息 |
| 验证文档类型| 用户提交的验证的姓名信息 |
| 时间| 用户提交的验证的时间 |


<Note>
系统仅根据Sumsub SDK配置展示部分关键字段，如需查看完整认证详情，请登录Sumsub控制台或联系KeySecure支持团队。
</Note>


## III. 支持操作说明

**1. 刷新验证状态**
功能位置：列表页 / 详情页右上角 > 【刷新状态】

操作说明：

系统将通过API实时同步KYC认证状态，并更新显示

若用户在App中完成验证，该刷新操作可立即反映最新状态

应用场景：用户反馈验证已完成但状态未更新时，可手动刷新

**2. 按条件筛选记录**
功能位置：列表页上方筛选栏

可支持筛选条件：

* 认证状态（已通过 / 未通过 / 审核中等）

* 提交时间范围

* 会员ID / 邮箱关键词

* 应用场景：快速定位特定会员或异常认证记录
