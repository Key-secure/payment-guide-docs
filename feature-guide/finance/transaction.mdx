---
title: 消费记录
---

本模块用于追踪和管理所有通过虚拟支付卡完成的交易记录，涵盖交易全生命周期的各类状态（授权、结算、退款等），帮助商户全面掌握每一笔卡片消费行为所产生的财务影响，并支持快速排查问题或对账分析。

## 功能概述
“消费记录”页面聚焦于C端用户通过平台发行的虚拟支付卡完成的真实交易行为数据，系统将自动记录交易明细、关联账户变动、手续费情况等内容，确保可追溯、可核对。

## 核心消费信息

**交易基础信息**

| 字段 | 描述 |
|------|------|
| 交易类型 | 授权 / 结算 / 退款  |
| 交易时间 | 交易发起时间（UTC+8 时间） |
| 交易状态 | 成功 / 失败 等状态 |
| 交易ID | 平台交易唯一编号 |
| 退款订单ID | 若消费有关联退款，记录对应退款订单ID |

<Warning>
请注意，部分退款交易无法与原交易建立绑定关系
</Warning>

**发起会员信息**

| 字段 | 描述 |
|------|------|
| 会员ID | 发起交易的用户唯一标识  |
| 会员邮箱 | 发起交易的用户注册邮箱  |
| 身份认证状态 | 如：认证通过   |


**卡片信息**
| 字段 | 描述 |
|------|------|
| 卡号（后四位） | 展示消费所使用的虚拟卡编号（脱敏）  |
| 卡片类型| 如：虚拟卡 / 实体卡  |


**消费详情**
| 字段 | 描述 |
|------|------|
| 消费金额 / 币种	 | 卡片消费金额及其对应的法币币种（如：100 USD）  |
| 手续费| 消费中产生的平台服务费、小额手续费等  |
| 消费商户名称| 实际交易发生的商户标识（如：Amazon / Netflix）  |
| 交易地国家 / 区域| 消费地所在国家，辅助判断是否为境外交易  |


## 账户变动记录

系统将同步展示交易带来的账户变化，包括：

**商户预算账户变动**：本次交易导致的预算账户扣款，涵盖交易金额与卡BIN费用


**会员资产变动**：展示因消费（含手续费）导致会员的数字货币资产减少情况 ，系统将按照支付优先级扣款。
