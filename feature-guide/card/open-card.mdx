## 功能概述

开卡记录”模块用于管理与审核所有 C 端用户的虚拟卡申请流程。商户可在此统一查看用户提交的所有开卡申请，并对待审批的申请进行审核操作。此模块有助于掌握用户申请行为、快速处理开卡流程及排查异常情况，是日常卡片运营与风控控制的关键环节之一。

## 核心功能

### 实时查看开卡申请列表

页面展示所有 C 端用户发起的开卡请求，支持筛选和排序，涵盖以下申请状态：

* 待审核：用户提交申请后，商户需手动审核通过或拒绝。

* 进行中：系统正在处理开卡请求，如创建卡片、绑定卡BIN等。

* 补充 KYC：用户需上传身份信息或补全资料，方可继续开卡流程。

* 成功：卡片已成功发放，用户可在 App 内正常使用。

* 失败：开卡流程中断或被用户撤销，未生成可用卡片。

支持通过筛选器按申请状态、时间、用户邮箱、商户 ID 等条件精准检索记录。

### 审核开卡申请

对处于“待审核”状态的记录，商户可进行如下操作：

* 通过申请：系统将进入下一步开卡流程，自动完成卡片创建流程。

* 拒绝申请：终止当前申请流程，用户将收到拒绝通知（可设置拒绝原因）。

### 查看申请详情

点击任一申请记录的【详情】按钮，可进入申请详情页，查看完整开卡数据，包括：

**基础信息**

* 字段

* 商户ID	

* 申请时间	

* 状态	

* 失败原因（如果失败则有失败原因）	

* 交易 ID	

**用户信息**
用户邮箱

会员 ID	

身份认证状态	

<Tip> 
目前系统只支持身份认证通过的会员申请开卡。
</Tip>

**卡片信息**

* 卡片类型：虚拟卡或实体卡

* App 开卡套餐	

* 会员开卡成本	

* 会员消费费率（境内）	

* 会员消费费率（境外）	

* 小额手续费（单笔）

* 小额手续费豁免（次数）	

* 小额手续费豁免周期


## 建议配置与注意事项

1.审核流程建议设置 自动 + 人工结合机制，提高效率并保留风险控制余地。

2.出现大量“补充 KYC”状态申请时，建议同步优化身份认证流程或前端引导文案。

3.可配合“卡BIN设置”、“开卡套餐”、“预算账户”使用，联动调控资金消耗与风控策略。

4.建议定期导出开卡数据以做用户行为分析和市场反馈复盘。
