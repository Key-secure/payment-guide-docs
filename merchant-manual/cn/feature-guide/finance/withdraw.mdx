---
title: 提现记录
---

## I. 功能概述

展示所有会员发起的提现请求，包括当前状态、处理进度与链上执行详情。每条记录包含以下信息：

| 字段 | 描述 |
|------|------|
| 发起人| 发起提现请求的会员唯一编号 |
| 币种 | 提现的加密货币和对应的链 |
| 数量 | 本次提现的真实转出数量 |
| 手续费数量 | 提现所扣除的手续费数量 |
| 提现数量 | 本次提现，客户请求转出的数量|
| To | 用户填写的提现收款钱包地址 |
| TXID | 成功提现后生成的链上交易哈希，用于链上查询与追溯 |
| 状态	 | 待审批 / 交易成功 / 审批拒绝 |

## II. 操作流程说明
提现遵循**“人工/自动审批 + MPC 钱包自动执行”**流程，保证出金动作安全、透明、可控。

<Steps>
<Step title="用户提交提现请求">
    用户在 支付App 或提交提现申请，系统会生成待处理记录，状态为 待审批。
</Step>
<Step title="后台人工审核">
后台运营人员登录控制台，访问「提现记录」模块，逐条审核申请：

通过：进入下一步，系统自动调用 MPC 钱包进行链上执行。

拒绝：申请终止，可填写拒绝原因（如 KYC 异常、提现地址不符等）。

</Step>
<Step title="系统自动发起链上提现">
审批通过后，系统会调用 MPC 钱包服务 发起链上交易；

提交成功后，通过 TXID（交易哈希），可查看链上详情。
</Step>
</Steps>

<Tip> 
1. 链上交易成功即视为提现完成，状态更新为“已通过”，否则系统将自动重试或记录失败状态。

2. Saas+商户，MPC 钱包归集功能必须启用，加油站钱包的矿工费必须充足。

3. 钱包服务离线或签名异常将直接影响提现流程。建议对接告警系统，配置状态监测与自动化通知。
</Tip>
