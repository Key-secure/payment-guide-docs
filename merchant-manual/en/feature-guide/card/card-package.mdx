## I. Feature Overview

The **App Card Issuance Packages** page is used to configure the card product packages that end-users can choose and apply for within the App. Merchants can create multiple user-facing card issuance packages based on business needs, cost structure, and target market, defining package prices, fee rules, and card service levels to achieve differentiated pricing, control margins, and improve user experience. Each card issuance package represents a sellable card product that users can choose when applying for a card in the App.

## II. Configuration Details

<Note>
Before creating a card issuance package, you must first set the currency to which refunded card balances are credited in the member account. Otherwise, package configuration cannot be completed.
</Note>

### Package Basic Information

Merchants need to configure clear, intuitive basic information for each package so that users can accurately understand the card product and its scope in the front-end App.

**Package Name**

The name shown in the App for the card (supports Chinese and English), e.g. “Basic Virtual Card”, “VIP Premium Card”.

**Package Description**

A detailed description of the card’s use cases, features, or fee information to help improve conversion.

**Package Tags**

Short labels for card type, e.g. “Basic”, “Premium”, “No Fee”, which can be shown as badges in the front-end.

Profitability tip:

To earn revenue from transactions, set the consumption fee rate higher than the card BIN fee rate. For example, if the card BIN rate is 2%, set the service fee to 7% (2% cost + 5% profit margin).
