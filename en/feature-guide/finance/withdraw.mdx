## 功能概述

“提现记录”页面用于管理 会员（C 端用户）发起的链上资产提现请求，支持审批操作与链上执行状态的全流程查看与追踪。该模块是数字资产出金过程中的核心环节，涉及人工审核、钱包签名及链上广播等关键步骤，确保资金安全与合规。

展示所有会员发起的提现请求，包括当前状态、处理进度与链上执行详情。每条记录包含以下信息：

| 字段 | 描述 |
|------|------|
| 会员 ID | 发起提现请求的会员唯一编号 |
| 币种 / 网络 | 提现的加密货币 |
| 提现金额| 本次请求提现的资产数量 |
| 提现地址 | 用户填写的收款钱包地址 |
| TXID | 成功提现后生成的链上交易哈希，用于链上查询与追溯 |
| 审批状态	 | 待审批 / 已通过 / 已拒绝 |
| 审批人| 进行人工审核的账户 |
| 审批时间| 进行人工审核的时间点 |


## 操作流程说明
提现申请遵循**“人工审批 + MPC 钱包自动执行”**流程，保证出金动作安全、透明、可控。

步骤 1：用户提交提现请求
用户在 支付App 或提交提现申请，系统会生成待处理记录，状态为 待审批。

步骤 2：后台人工审核
后台运营人员登录控制台，访问「提现记录」模块，逐条审核申请：

通过：进入下一步，系统自动调用 MPC 钱包进行链上执行。

拒绝：申请终止，可填写拒绝原因（如 KYC 异常、提现地址不符等）。

<Note>
建议结合会员身份认证状态、交易行为、异常检测系统等信息，辅助判断是否批准提现。

可以设置自动提现策略，低于一定价值数量的提现将自动审批通过。
</Note>


步骤 3：系统自动发起链上提现
审批通过后，系统会调用 MPC 钱包服务 发起链上交易；

多方签名机制将由系统协同完成（Co-signer）；

提交成功后，生成 TxID（交易哈希），可跳转查看链上详情。

<Note>
链上交易成功即视为提现完成，状态更新为“已通过”，否则系统将自动重试或记录失败状态。
</Note>


## 重要运行要求与提醒

**MPC 钱包归集功能必须启用**

系统调用钱包的出金逻辑依赖于自动化资产归集与地址管理，确保每笔出金请求链路完整。

**确保加油站地址矿工费充足**

所有链上提现将消耗 gas fee，建议定期为 MPC 钱包内的加油站账户补充原生代币（如 ETH、TRX 等）以避免出金失败。

**实时监控 MPC 钱包运行状态**

钱包服务离线或签名异常将直接影响提现流程。建议对接告警系统，配置状态监测与自动化通知。
